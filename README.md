# System Monitor – консольное приложение для мониторинга системы

## Техническое задание (ТЗ)

Разработайте консольное приложение на C# для мониторинга ключевых метрик системы, которые должны работать на Windows или на Linux. Приложение собирает данные о состоянии системы, отображает их в реальном времени с задаваемой периодичностью в консоль или сохраняет в логафай истории для последующего анализа.

Собираемые метрики:

CPU:
1. Общая загрузка (%).
2. Загрузка по каждому ядру (%).

Память:
1. Всего RAM (MB).
2. Доступно RAM (MB).
3. Использование (%).

Сеть:
1. Входящий трафик (байт/сек).
2. Исходящий трафик (байт/сек).

Конфигурирование приложения:
Приложение должно содержать файл конфигурации в формате на выбор разработчика, например в json.
Файл конфигурации обязательно содержит:
1. Периодичность в N секунд вывода информации.
2. Куда выводится информация, а именно, в консоль или в лог файл.
Разработчик может сам добавить дополнительные поля, которые он сочтет нужным.

Технические требования:
1. Платформа: .Net 6/7/8/9.
2. Язык: C#.
3. Поддерживаемые ОС: Windows или Linux (Ubuntu или Debian).

Оформление проекта:
1. Проект нужно опубликовать на GitHub.
2. Сборщики метрик для CPU, памяти и сети должны быть реализованы в отдельных классах, унаследованных от базового.

## Реализация

Поддерживаются Windows и Linux (включая WSL). Все метрики собираются в реальном времени, выводятся в консоль или файл согласно настройкам `config.json`.

## Пример содержимого **config.json**

```json
  "period_seconds": 2,
  "output_mode": "Console",
  "log_file_path": "metrics.log"
```
- period_seconds - Периодичность вывода (сек)
- output_mode - Куда выводить данные (File или Console)
- log_file_path - Путь к лог-файлу

### Основные возможности

- **CPU** – общая загрузка и загрузка по каждому ядру (через `PerformanceCounter` на Windows, `/proc/stat` на Linux).
- **Память** – общая, доступная, процент использования (через `GlobalMemoryStatusEx` на Windows, `/proc/meminfo` на Linux).
- **Сеть** – входящая и исходящая скорость (байт/с) – кроссплатформенно через `NetworkInterface`.
- **Конфигурация** – JSON-файл `config.json` (создаётся автоматически).
- **Вывод** – в консоль или в лог-файл (с автоматическим созданием папок).
- **Кроссплатформенность** – условная компиляция (`#if WINDOWS` / `#elif LINUX`).

### Структура проекта

- `MetricCollector.cs` – абстрактный базовый класс.
- `CpuCollector.cs`, `MemoryCollector.cs`, `NetworkCollector.cs` – фасады.
- `*Windows.cs` – реализации для Windows.
- `*Linux.cs` – реализации для Linux.
- `Configuration.cs` – загрузка/сохранение JSON-конфига.
- `Program.cs` – основной цикл и логика вывода.
